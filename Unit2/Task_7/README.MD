#### Задача 7: Graceful shutdown через context (тик-стрим + воркеры)

**Краткое описание и цель навыка**

Нам дано: поток “тик” событий (каждый тик — число). У нас есть `N` воркеров, которые обрабатывают тики параллельно.  
Нужно: **по сигналу cancel** остановить обработку без утечек и зависаний — то есть:
- продьюсер перестаёт слать новые тики,
- воркеры корректно завершаются,
- результаты аккуратно собираются,
- программа всегда завершает работу.
  
Чтобы сделать — вспомни: `context.WithCancel`, `ctx.Done()`, `select`, закрытие каналов, `WaitGroup`.

**Условие**
- `M` — сколько тиков всего возможно,
- `N` — число воркеров,
- `K` — момент отмены (после отправки **ровно K тиков** контекст отменяется),
- массив из M чисел — значения тиков.
  
Правила потока:
- продьюсер отправляет тики по порядку из массива,
- как только отправлено `K` тиков (или если `K > M`, то после всех `M`) — вызывается `cancel()` и продьюсер завершает работу,
- воркеры должны обрабатывать только те тики, которые успели прийти до cancel.

Обработка тика: `f(x) = x * x` (квадрат).  
Нужно вывести:
- сколько тиков обработано
- сумму квадратов обработанных тиков
