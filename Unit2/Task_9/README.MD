#### 2.1 Горутины и каналы

**Задача 9: Неблокирующая отправка и чтение из канала**

**Краткое описание и цель навыка**

Нам дано: **буферизированный канал** и значение `v`.
Нужно: попытаться **отправить** `v` в канал так, чтобы **не блокироваться**, если канал занят.
Потом попытаться **прочитать** из канала так, чтобы** не блокироваться**, если канал пустой.

Ключ:
- `select { case ch <- v: ... default: ... }`
- `select { case x := <-ch: ... default: ... }`

Чтобы сделать — вспомни: `default` в `select` делает операцию **неблокирующей**.

**Условие**

Нам дано:
- ёмкость канала `B`,
- сколько элементов уже лежит в канале `S` `(0 ≤ S ≤ B)`,
- значение `v`,
- затем `S` чисел — они будут заранее отправлены в канал (в этом порядке).

Напиши:
- `TrySend(ch, v) bool` — пытается отправить `v` **неблокирующе**. Возвращает `true`, если отправка прошла, иначе `false`.
- `TryRecv(ch) (int, bool)` — пытается прочитать **неблокирующе**. Возвращает `(value, true)`, если удалось прочитать, иначе `(0, false)`.

После этого программа должна:
1. вызвать `TrySend(ch, v)`
2. вызвать `TryRecv(ch)`
и вывести результат.

**Формат ввода**

- Первая строка: `B` `S` `v`
- Вторая строка: `S` целых чисел (может отсутствовать, если `S = 0`)

Тестовый данные

|Input|Output|
|:--|:--|
|48 12 -237 | 1 -754
-754 321 -550 -607 522 267 853 994 958 129 593 -534
